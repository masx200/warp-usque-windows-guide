# Usque：开源 Cloudflare WARP MASQUE 协议实现来了

在当今互联网时代，网络隐私安全和访问自由度已成为越来越多用户关注的核心议题。Cloudflare
推出的 WARP
服务以其简洁的设计和强大的功能赢得了全球用户的青睐，然而官方客户端在某些场景下显得不够灵活，尤其是对于需要自定义代理方案的用户而言。值得庆幸的是，开源社区从未停止探索的脚步——今天我要为大家介绍的
**Usque**，正是这样一个极具创新性的开源项目，它是 Cloudflare WARP 客户端 MASQUE
模式的开源重实现，为用户提供了一个轻量、灵活且完全可控的 WARP 解决方案。

## 初识 Usque：什么是 MASQUE 协议

在深入了解 Usque 之前，我们首先需要理解它所依赖的核心技术——MASQUE
协议。MASQUE（Multiplexed Application Substrate over QUIC Encryption）是一种基于
QUIC 协议的新型代理协议，它充分利用了 QUIC 协议的 multiplex
能力，能够在单一连接上同时处理多个数据流。MASQUE
协议的核心优势在于其流量混淆能力：与传统的 VPN 协议不同，MASQUE 流量与普通的
HTTP/3 流量几乎无法区分，这为用户在网络审查严格的环境下提供了更好的保护。

Usque 项目基于 RFC 9484 定义的 Connect-IP
协议，这是一个相对较新的标准，专门用于在 MASQUE 架构中实现 IP
隧道功能。项目作者Diniboy1123在研究过程中发现，现有的开源实现大多不够完善，因此决定自己动手，从零开始构建这个项目。从技术架构角度来看，Usque
完全使用 Go 语言编写，依赖于成熟的 quic-go 库来实现 QUIC
协议栈，这使得项目能够轻松实现跨平台支持——无论是 Windows、Linux、macOS，还是
Android 和 iOS，都能找到适合的运行方式。

## Usque 的核心特性与优势

Usque 作为一款开源的 WARP
客户端替代方案，其设计理念围绕着「轻量」与「灵活」两个核心关键词展开。与
Cloudflare 官方客户端相比，Usque
具备诸多显著优势，这些优势使其成为技术爱好者和有特殊需求用户的理想选择。

首先，从开源角度来看，Usque
最大的价值在于其完全透明。任何人都可以审查其源代码，了解其工作原理，这对于关注网络安全的用户来说尤为重要。官方客户端是一个黑盒系统，用户无法得知其具体实现了哪些功能、收集了哪些数据。而
Usque
的代码完全开放，用户可以放心地将其部署在生产环境中，无需担心隐藏的后门或数据收集行为。

其次，在资源占用方面，Usque 表现出色。官方客户端在 Android
平台上的安装包体积高达 260MB
以上，这对于存储空间有限的移动设备用户而言是一个不小的负担。相比之下，Usque
只是一个几十 MB 的 Go
二进制文件，部署简单且运行高效。根据项目作者的测试数据，在使用 SOCKS5
代理模式时，Usque 能够跑满 1Gbps 的带宽，下行速度达到 833.60Mbps，上行速度达到
772.88Mbps，而 CPU 占用率仅约 26%，这样的性能表现可以说是相当令人满意的。

在功能模式方面，Usque 提供了四种不同的工作模式以适应不同的使用场景。第一种是
SOCKS5 代理模式，这是最简单易用的模式，跨平台且无需 root
权限，非常适合快速部署和临时使用。第二种是 HTTP 代理模式，与 SOCKS5
类似但兼容性更广，几乎所有客户端都能支持。第三种是原生 TUN
隧道模式，这是效率最高的模式，能够直接创建虚拟网络接口，适合需要完整 VPN
体验的高级用户。第四种是端口转发模式，支持 WARP to WARP
设备间通信，适用于企业级应用场景。

## 在 Windows 上部署 Usque：完整指南

了解了 Usque 的强大功能之后，接下来让我们看看如何在 Windows
系统上部署这个项目。根据仓库中的详细指南，部署过程主要分为以下几个步骤，每个步骤都需要用户仔细操作以确保最终能够顺利运行。

### 准备工作与环境配置

在开始部署之前，我们需要确保系统满足以下要求：操作系统建议使用 Windows 10 或
Windows 11，PowerShell 版本需要在 5.1
或更高版本，还需要稳定的互联网连接。此外，我们还需要下载 NSSM（Non-Sucking
Service Manager）这款工具，它的作用是将 PowerShell 脚本注册为 Windows
服务，从而实现开机自启动和故障自动重启等功能。

具体的环境配置步骤如下：首先创建项目目录，在 PowerShell 中执行
`New-Item -ItemType Directory -Path "C:\WARP-Usque" -Force`
命令创建工作目录。然后访问 GitHub Releases 页面下载最新版本的 Usque Windows
amd64 二进制文件，解压到刚才创建的目录中。接下来下载并解压 NSSM
工具，建议将其放置在 `C:\Tools\nssm` 目录下以便于管理。如果系统中已经安装了
Chocolatey 包管理器，也可以直接使用 `choco install nssm` 命令来快速安装 NSSM。

### 注册 Cloudflare WARP 账户

完成环境配置后，第二步是注册 Cloudflare WARP 账户。Usque
提供了便捷的注册命令，可以自动完成账户创建和设备注册的全过程。对于普通用户，可以使用以下命令创建新账户并注册设备：`.\usque.exe register -a`。如果需要为设备指定名称，可以添加
`-n` 参数，例如 `.\usque.exe register -a -n "MyWindowsPC"`。对于使用 Cloudflare
Zero Trust 的企业用户，则需要提供 JWT token 进行注册，命令格式为
`.\usque.exe register --jwt "your-jwt-token"`。

注册成功后，Usque 会在当前目录下生成一个 `config.json` 文件，这个文件包含了连接
WARP 网络所需的全部配置信息，包括设备私钥、服务器端点地址、许可证信息以及设备 ID
等。妥善保管这个配置文件非常重要，因为它相当于设备的身份凭证，每次启动代理时都会读取其中的信息。

### 启动 SOCKS5 代理服务

账户注册完成后，就可以启动 SOCKS5 代理服务了。最简单的启动方式是直接执行
`.\usque.exe socks` 命令，这会使用默认配置启动一个无认证的 SOCKS5
代理。然而，为了安全起见，我们强烈建议配置用户名和密码进行认证。

一个完整的启动命令示例如下：`.\usque.exe socks -b 0.0.0.0 -p 1080 --username "your-username" --password "your-password" -d 1.1.1.1 -d 8.8.8.8 -d 9.9.9.9 -d 94.140.14.14 -s www.cloudflare-gateway.com`。其中
`-b` 参数指定绑定地址，`0.0.0.0` 表示接受来自所有网络接口的连接；`-p`
参数指定监听端口，默认是 1080；`-d` 参数用于指定 DNS
服务器，可以多次使用以配置多个 DNS；`-s` 参数用于指定 SNI，对于 Zero Trust
用户需要设置为 `zt-masque.cloudflareclient.com`。

### 配置 Windows 服务实现开机自启动

为了确保代理服务能够持续稳定运行并实现开机自启动，我们需要将其配置为 Windows
系统服务。这需要使用 NSSM 工具来完成，首先需要创建一个 PowerShell
启动脚本，在脚本中配置好所有必要的参数，然后将这个脚本注册为系统服务。

启动脚本的核心逻辑是配置好代理服务的各项参数，然后调用 Usque 的 SOCKS5
模式。配置完成后，使用 NSSM 安装服务的命令格式如下：首先执行
`$NSSM_PATH install usque "pwsh.exe"` 来注册服务，然后通过一系列
`$NSSM_PATH set usque ...`
命令来配置服务的各项参数，包括启动脚本路径、工作目录、日志输出位置、服务描述等，最后执行
`$NSSM_PATH start usque`
来启动服务。配置成功后，服务就会在系统启动时自动运行，即使遇到崩溃也会自动重启。

## 连接监控与保活机制

在实际运行过程中，Usque
可能会遇到一些稳定性问题，比如长时间无数据传输时连接会自动断开，或者网络波动导致频繁重连。为了解决这些问题，仓库中提供了完善的监控与保活解决方案。

监控脚本的工作原理是定期执行三项任务：第一是发送保活流量，通过定期访问
Cloudflare
的接口来保持连接的活跃状态，防止因长时间无活动而被服务器断开；第二是测试代理连接，通过多个不同的
API 接口来验证代理是否正常工作，并解析返回的 IP
信息来判断当前的归属地；第三是自动恢复，当检测到连接持续失败达到一定次数后，会自动重启
Usque 服务以恢复正常运行。

监控脚本会生成详细的日志文件，记录每一次检查的结果和所有重要事件。日志文件按日期命名，便于用户进行历史查询。此外，脚本还会生成一个
`status.json` 文件，其中包含当前的连接状态、IP 地址、归属国家和 ISP
信息，用户可以基于这个文件构建自己的监控面板。

## 进阶配置与性能优化

对于追求更佳体验的用户，Usque 还提供了一些进阶配置选项。在 DNS
优化方面，合理选择 DNS
服务器可以显著提升响应速度。项目作者推荐使用以下组合：Cloudflare 的
1.1.1.1（最快）、Google 的 8.8.8.8、Quad9 的 9.9.9.9（注重隐私保护）以及 AdGuard
的 94.140.14.14（具备广告拦截功能）。在启动命令中通过多次使用 `-d`
参数来指定这些 DNS 服务器即可。

在企业应用场景下，Usque 也支持 Cloudflare Zero Trust
的集成。用户只需要在注册时提供 Zero Trust 的 JWT token，并在启动时将 SNI 设置为
`zt-masque.cloudflareclient.com`，即可连接到企业的 Zero Trust
网络，享受更加完善的管理和安全策略。此外，Usque 还支持多实例部署，通过 `-c`
参数指定不同的配置文件，可以在同一台机器上运行多个独立的代理实例，每个实例使用不同的端口和账户。

## 总结与展望

Usque 项目的出现为广大用户提供了一个全新的 WARP
使用方式。它以开源、轻量、灵活的特性打破了官方客户端的诸多限制，让用户能够更加自主地控制自己的网络代理方案。无论是想要构建稳定
SOCKS5 代理服务的个人用户，还是需要定制化解决方案的企业用户，Usque 都值得一试。

当然，作为一款相对年轻的开源项目，Usque
仍有不少可以改进的空间。比如性能方面仍有优化余地，Windows 平台上的原生 TUN
隧道模式支持也还在开发中。但正是这种开源精神推动着项目不断进步，我们期待着未来能看到更加完善的
Usque，为用户的网络体验带来更多可能性。
