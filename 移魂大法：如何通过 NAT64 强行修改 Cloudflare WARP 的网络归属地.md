# 🚀 告别付费 VPN！手把手教你用 NAT64 物理“漂移” Cloudflare WARP 归属地

你是不是还在忍受 Cloudflare WARP 默认分配的“就近” IP？明明想要看美区
Netflix，结果 WARP 偏偏把你分配到了香港或者新加坡？

今天分享一个**进阶级黑科技**：无需购买昂贵的 VPN，只需利用 **NAT64
端点（Endpoint）**，就能强行把你的 WARP 网络出口改到**美国**或**荷兰**！

---

## 💡 为什么 WARP 的地区总是不对？

Cloudflare WARP 使用的是 **Anycast（任播）**
技术。默认情况下，你的设备会连接到物理距离最近的 Cloudflare 数据中心。

- 你在广东，出口就是香港；
- 你在上海，出口就是东京。

**核心逻辑：** 只要我们能让 WARP
认为我们是从“美国机房”发起的连接，它就会乖乖地给我们分配一个美国的出口 IP。而
**NAT64** 正是实现这一跨时空连接的“虫洞”。

---

## 🛠️ 准备工作：NAT64 前缀列表

感谢 [BPB-Worker-Panel](https://github.com/bia-pain-bache/BPB-Worker-Panel)
整理的公开 NAT64 资源，这是我们改变归属地的关键坐标：

| 🌍 目标地区           | 📍 NAT64 前缀 (Prefix) |
| --------------------- | ---------------------- |
| **荷兰 (Netherland)** | `[2a02:898:146:64::]`  |
| **美国 (USA - A)**    | `[2602:fc59:b0:64::]`  |
| **美国 (USA - B)**    | `[2602:fc59:11:64::]`  |

---

## 📝 实操步骤：三步完成“大挪移”

### 第一步：挑选一个 Cloudflare IPv4 地址

你可以使用 Cloudflare 默认的任一端点 IP，例如：`162.159.199.2`。

### 第二步：合成你的“专属”端点地址

我们需要把上面的 IPv4 地址嵌入到 NAT64 的前缀中。这里有两种写法：

1. **简单写法（部分客户端支持）：** 直接拼接：`[2602:fc59:b0:64::162.159.199.2]`
2. **标准写法（十六进制）：** 将 `162.159.199.2` 转换成十六进制后拼接。

> **示例（美国区）：** `[2602:fc59:b0:64::a29f:c702]`

### 第三步：修改 WARP 配置

打开你的 WARP 客户端（或者 WireGuard / V2Ray 等工具），找到 **Endpoint（端点）**
设置，将原本的地址替换为我们合成后的地址。

- **Endpoint:** `[2602:fc59:b0:64::a29f:c702]:2408`

保存并重新连接，奇迹发生了——你的归属地现在变成了 **USA**！

---

## 🧐 原理拆解

NAT64 的本质是一个“翻译官”。当你连接美国的前缀时，你的流量会先通过 IPv6
隧道跑到位于美国的 NAT64 网关，然后再从那里连接到 Cloudflare 的网络。Cloudflare
看到请求来自美国机房，自然就会给你分配当地的 IP 出口。

---

## ⚠️ 避坑小贴士

- **网络要求：** 你的本地网络环境必须支持
  **IPv6**（或者你的代理软件能够解析并处理 IPv6 流量）。
- **延迟波动：** 因为流量绕道了国外的 NAT64 网关，延迟（Ping
  值）会有所增加，但在刷流媒体和网页时，这种增加几乎感觉不到。
- **稳定性：** 公开的 NAT64
  前缀可能会有负载限制，如果连接不上，可以尝试换一个前缀。
